设计模式（03）代理模式

发表于 2018-01-12 | 分类于 设计模式

基本概念

    1. 什么是代理模式？
    • 代理模式的作用是：为其他对象提供一种代理以控制对这个对象的访问。

  2. 代理模式有什么好处？

    • 在某些情况下，一个客户不想或者不能直接引用另一个对象，而代理对象可以在客户端和目标对象之间起到中介的作用。
  3. 代理模式一般涉及到的角色有：
    • 抽象角色：声明真实对象和代理对象的共同接口；
    • 代理角色：代理对象角色内部含有对真实对象的引用，从而可以操作真实对象，同时代理对象提供与真实对象相同的接口以便在任何时刻都能代替真实对象。同时，代理对象可以在执行真实对象操作时，附加其他的操作，相当于对真实对象进行封装。
    • 真实角色：代理角色所代表的真实对象，是我们最终要引用的对象。
  4. UML类图
    • RealSubject 是委托类，Proxy 是代理类；
    • Subject 是委托类和代理类的接口；
    • request() 是委托类和代理类的共同方法

      静态代理

      静态代理实现中，一个委托类对应一个代理类，代理类在编译期间就已经确定

      动态代理

      动态代理中，代理类并不是在Java代码中实现，而是在运行时期生成，相比静态代理，动态代理可以很方便的对委托类的方法进行统一处理，如添加方法调用次数、添加日志功能等等，动态代理分为jdk动态代理和cglib动态代理

代码示例 https://github.com/iycc/pattern-demo/tree/master/pattern-proxy

设计模式（02）策略模式

发表于 2018-01-10 | 分类于 设计模式
  1. 策略模式概念
    • 封装变化
    • 面向接口编程
  2. 策略模式的使用场景
    • 算法可封装起来，根据不同场景调用不同的算法
  3. 涉及到的角色
    • 抽象策略角色：策略类，通常由一个接口或者抽象类实现
    • 具体策略角色：包装了相关的算法和行为
    • 环境角色：持有一个策略类的引用，最终给客户端调用的。
  4. JDK中运用策略模式的实例
    • Java.util.Collections#sort(List list, Comparator < ? super T > c)
    • java.util.Arrays#sort(T[], Comparator < ? super T > c)
    • Java正则表达式Pattern
    • ThreadPoolExecutor中的ThreadFactory和RejectedExecutionHandler
    • hashmap中对hashcode的计算
    • runnable与executor
    • recursive和forkjoin
    • SWT Layout
  5. 策略模式的缺点
    • 客户端必须知道所有的策略类，并自行决定使用哪一个策略类。
    • 造成很多的策略类。

参考 http://www.importnew.com/12853.html
